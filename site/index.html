What is OminiConnect

The easiest way to give AI agents
access to real-world APIs

Most AI projects spend weeks building integrations. OminiConnect solves this by providing one unified API layer on top of 700+ services — GitHub, Slack, Notion, Salesforce, and more.

API Proxy

Call any provider API through OminiConnect using the same interface. Handles auth headers, token refresh, and response parsing automatically.

  • Direct REST calls to 700+ providers
  • Automatic OAuth token management
  • Consistent response format across all APIs

Tool Registry

Discover what each provider can do through a standardized tool schema. Each tool has a clear description, input parameters, and required OAuth scopes.

  • Standardized tool definitions per provider
  • Scope satisfied check before execution
  • MCP-compatible (Model Context Protocol)

AI Delegation

Send natural language requests and let OminiConnect's LLM choose the right tools, execute them in sequence, and return the final result.

  • Natural language interface
  • Automatic multi-step orchestration
  • Built on LiteLLM for provider flexibility

Credential & Token Manager

Manage OAuth2 tokens, API keys, and Basic auth credentials in one secure vault. Tokens are refreshed automatically — no more expired credentials breaking your workflows.

  • OAuth2, API Key, Basic auth supported
  • Automatic token refresh — no more expired credentials
  • Secure token vault with auto-refresh

📖 Code examples — Python, JavaScript, Go, Rust, Flutter, REST API

🐍 Python — setup & list connectors

from ominiconnect import OminiConnect

client = OminiConnect(api_key="sk_...")

connectors = client.connectors.list()
print(connectors)

🐍 Python — execute a tool

result = client.tools.execute(
    "github_list_repos",
    arguments={"limit": 10}
)
print(result["body"])

📜 JavaScript — setup & list connectors

import { OminiConnect } from '@ominiconnect/sdk';
const client = new OminiConnect({ apiKey: 'sk_...' });
const connectors = await client.connectors.list();
console.log(connectors);

📜 JavaScript — execute a tool

const result = await client.tools.execute(
    'github_list_repos',
    { limit: 10 }
);
console.log(result.body);

🔵 Go — call an API directly

client := ominiconnect.New("sk_...", nil)
connectors, _ := client.Connectors.List()
fmt.Println(connectors)

🦀 Rust — setup & list connectors

let client = Client::new("sk_...", None);
let connectors = client.connectors().list().await?;
println!("{:?}", connectors);

🦋 Flutter — setup & list connectors

import 'package:ominiconnect/ominiconnect.dart';
final client = OminiConnect(apiKey: 'sk_...');
final connectors = await client.connectors.list();

🌐 REST — tools/execute

Natural language API

Tell OminiConnect what you want — it picks the right tool

No more figuring out which API endpoint to call. Just describe what you want in plain English — OminiConnect selects the right tool, passes the right arguments, and returns the result.

🗣️

Natural language input

"list my GitHub repos sorted by last updated"

⚙️

OminiConnect picks the tool

Tool: github_list_repos with args {sort: "updated"}

Python SDK

resp = client.llm.execute(
    "list my GitHub repos sorted by last updated"
)
print(resp.result)
# {'ok': True, 'body': [...], 'tool': 'github_list_repos'}

JavaScript SDK

const resp = await client.llm.execute(
    'list my GitHub repos sorted by last updated'
);
console.log(resp.result);
// {ok: true, body: [...], tool: 'github_list_repos'}

REST API

POST /api/llm
{
  "query": "list my GitHub repos sorted by last updated"
}
// Response: {ok: true, tool: "github_list_repos", result: {...}}

🦀 Rust — natural language execution

    .execute("list my repos", None)

🌐 cURL — natural language request

curl -X POST http://localhost:9000/api/llm \
  -H "Authorization: Bearer sk_..." \
  -H "Content-Type: application/json" \
  -d '{"query": "list my GitHub repos"}'

MCP — Tool discovery

Or let your AI agent discover tools via MCP

OminiConnect speaks MCP (Model Context Protocol). Any MCP-compatible AI agent can discover and call your connected tools automatically.

Step 1 — Agent asks: what tools do I have?

// POST /api/mcp { "jsonrpc": "2.0", "id": 1, "method": "tools/list", "params": {} } // Returns: list of available tools with descriptions & input schemas

Step 2 — Agent calls a tool

// POST /api/mcp { "jsonrpc": "2.0", "id": 2, "method": "tools/call", "params": { "name": "github_list_repos", "arguments": {"sort": "updated", "per_page": 5} } } // Response: tool result with repository data

Quick start

Start using OminiConnect

Get your API key, connect a service, make your first call — all in under 5 minutes.

1

Sign up and get an API key

Create an account at the OminiConnect portal and copy your API key.

2

Connect a provider

Go to the Integrations page, select a provider (e.g. GitHub), and complete OAuth or enter an API key.

3

Make your first API call

Use the REST API or SDK to call the provider through OminiConnect.

4

Connect your AI agent

Point your AI agent to OminiConnect's MCP endpoint — it will discover and use your connected tools automatically.

Python — list connectors

from ominiconnect import OminiConnect

client = OminiConnect(api_key="sk_...")
connectors = client.connectors.list()
print(connectors)

Python — execute tool

result = client.tools.execute(
    "github_list_repos",
    platform="github",
    arguments={"limit": 10}
)
print(result.body)

cURL — direct proxy call

curl -X POST https://api.ominiconnect.com/api/proxy/github \
  -H "Authorization: Bearer sk_..." \
  -H "Content-Type: application/json" \
  -d '{"endpoint": "/user/repos", "method": "GET"}'
