OminiConnect 是什么

让 AI 代理轻松访问
真实世界 API 的最简方案

大多数 AI 项目在集成第三方服务上花费数周时间。OminiConnect 通过在 700+ 服务之上提供统一 API 层来解决这个问题——GitHub、Slack、Notion、Salesforce 等等。

API 代理

通过 OminiConnect 调用任何服务商 API，使用统一接口。自动处理认证、Token 刷新和响应解析。

  • 直接 REST 调用 700+ 服务商
  • 自动 OAuth Token 管理
  • 跨服务商响应格式统一

工具注册表

通过标准化工具架构发现每个服务商的功能。每个工具有清晰的描述、输入参数和所需 OAuth 范围。

  • 每个服务商标准化工具定义
  • 执行前检查 OAuth 范围
  • 兼容 MCP（模型上下文协议）

AI 委托

发送自然语言请求，由 OminiConnect 的 LLM 选择合适工具、依次执行并返回最终结果。

  • 自然语言交互接口
  • 自动多步骤编排
  • 基于 LiteLLM，灵活切换模型

凭证与 Token 管理器

在一个安全的保管库中管理 OAuth2 令牌、API 密钥和 Basic 认证凭证。令牌自动刷新——不再有凭证过期导致工作流中断。

  • 支持 OAuth2、API 密钥、Basic 认证
  • 自动刷新 Token，凭证永不过期
  • 基于 Nango，支持 700+ 服务商

📖 代码示例 — Python、JavaScript、Go、Rust、Flutter、REST API

🐍 Python — 初始化 & 列出已连接服务商

from ominiconnect import OminiConnect

client = OminiConnect(api_key="sk_...")

connectors = client.connectors.list()
print(connectors)

🐍 Python — 执行工具

result = client.tools.execute(
    "github_list_repos",
    arguments={"limit": 10}
)
print(result["body"])

📜 JavaScript — 初始化 & 列出已连接服务商

import { OminiConnect } from '@ominiconnect/sdk';
const client = new OminiConnect({ apiKey: 'sk_...' });
const connectors = await client.connectors.list();
console.log(connectors);

const result = await client.tools.execute(
    'github_list_repos',
    { limit: 10 }
);
console.log(result.body);

🔵 Go — 调用 API（Maton 风格）

🦀 Rust — 初始化 & 列出已连接服务商

🦋 Flutter — 初始化 & 列出已连接服务商

自然语言 API

告诉 OminiConnect 你想要什么——它自动选择工具

无需操心该调用哪个 API 端点。只需用自然语言描述你的需求——OminiConnect 会选择正确的工具、传入正确的参数、返回结果。

🗣️

自然语言输入

"列出我最近更新的 GitHub 仓库"

⚙️

OminiConnect 选择工具

工具: github_list_repos 参数 {sort: "updated"}

🐍 Python SDK

resp = client.llm.execute(
    "列出我最近更新的 GitHub 仓库"
)
print(resp.result)
# {'ok': True, 'body': [...], 'tool': 'github_list_repos'}

📜 JavaScript SDK

const resp = await client.llm.execute(
    '列出我最近更新的 GitHub 仓库'
);
console.log(resp.result);
// {ok: true, body: [...], tool: 'github_list_repos'}

🌐 REST API

POST /api/llm
{
  "query": "列出我最近更新的 GitHub 仓库"
}
// 响应: {ok: true, tool: "github_list_repos", result: {...}}

MCP — 工具发现

或让 AI 代理通过 MCP 自动发现工具

OminiConnect 支持 MCP（模型上下文协议）。任何 MCP 兼容的 AI 代理都能自动发现并调用你连接的工具。

步骤 1 — 代理询问：我的工具有哪些？

// POST /api/mcp { "jsonrpc": "2.0", "id": 1, "method": "tools/list", "params": {} } // 返回：可用工具列表，包含描述和输入模式

步骤 2 — 代理调用工具

// POST /api/mcp { "jsonrpc": "2.0", "id": 2, "method": "tools/call", "params": { "name": "github_list_repos", "arguments": {"sort": "updated", "per_page": 5} } } // 响应：仓库数据

与你使用的 AI 代理集成

Claude Desktop MCP 原生

将 OminiConnect 添加到 Claude Desktop 配置，它会自动将你连接的所有平台发现为工具。

Cursor / Windsurf MCP 原生

将 Cursor 或 Windsurf 指向 OminiConnect 的 MCP 端点，在 AI 自动补全和聊天中使用你的已连接工具。

任意 AI 框架 LangChain · LlamaIndex · 自定义

通过 REST 从任何 AI 框架调用 tools/execute 或 llm/execute。MCP 客户端库也可以原生使用。

快速开始

开始使用 OminiConnect

获取 API Key → 连接服务商 → 发出第一个请求——全部在 5 分钟内完成。

1

注册并获取 API Key

在 OminiConnect 门户注册账号，复制你的 API Key。

2

连接一个服务商

进入 Integrations 页面，选择服务商（如 GitHub），完成 OAuth 或输入 API 密钥。

3

发出第一个 API 调用

使用 REST API 或 SDK 通过 OminiConnect 调用该服务商。

4

连接你的 AI 代理

将 AI 代理指向 OminiConnect 的 MCP 端点——它会自动发现并使用你已连接的工具。

Python — 列出已连接的服务商

from ominiconnect import OminiConnect

client = OminiConnect(api_key="sk_...")
connectors = client.connectors.list()
print(connectors)

Python — 执行工具

result = client.tools.execute(
    "github_list_repos",
    platform="github",
    arguments={"limit": 10}
)
print(result.body)

cURL — 直接代理调用

curl -X POST https://api.ominiconnect.com/api/proxy/github \
  -H "Authorization: Bearer sk_..." \
  -H "Content-Type: application/json" \
  -d '{"endpoint": "/user/repos", "method": "GET"}'
